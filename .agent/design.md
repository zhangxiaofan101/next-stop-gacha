# Design — 下一站，去哪玩

> 本文件是架构的**现在时**：机制、不变式、模块注册。无状态、无日期——时间线见 state.md，出处见 git blame。设计可先行于代码：注册表中未落地模块的条目即其 spec of record。

## 架构（四层）

```
┌────────────────────────────────────────────────────────────┐
│ 展示层  主题皮肤系统（多皮肤 token + 插画资产层）            │
│         组件化 UI：筛选台 · 卡片网格 · 详情 · 对比 · 扭蛋机  │
│         行程单 · 路书视图 · 足迹地图                         │
├────────────────────────────────────────────────────────────┤
│ 决策层  过滤/排序引擎 · 条件随机 · 顺路排序(贪心+最小绕路)   │
│         交通启发式 · 预算估算 · 路书装配 · 分享合并          │
│         —— 纯函数模块，Vitest 单测覆盖                       │
├────────────────────────────────────────────────────────────┤
│ 数据层  六区 JSON + 补丁 + routes（仓库 data/ 唯一真相源）   │
│         构建校验闸门 → 发布为静态数据 chunk，运行时按区 fetch │
├────────────────────────────────────────────────────────────┤
│ 边缘层  Cloudflare Worker：路径前缀剥离 + 静态资产服务        │
│         + /api（KV：短链分享 · Durable Object：同步码+限流） │
│         免费额度内运行                                       │
└────────────────────────────────────────────────────────────┘
```

**前端技术栈**：Vite + TypeScript，零运行时框架。展示层为按视图切分的模板渲染模块（每个视图模块独占一块 DOM 区域及其事件接线，配合文档级事件委托）——本应用没有高频细粒度状态同步需求，模板字符串渲染在两模型家族都是低出错率习语，视图模块天然承载组件级样式（手绘换肤的挂点），不引入组件框架（Preact 已评估否决：等价重写全部渲染路径只添回归风险、无用户可感知收益，拍板记录见 state 档案）。决策层与展示层严格分离——决策层为无 DOM 依赖的纯函数模块，是单测的主战场；`tsc --noEmit` 与决策层单测同为构建门禁，DOM/网络副作用只存在于视图模块与服务层（天气 I/O）。

**数据外置**：目的地/线路数据不再注入 HTML，构建时经校验后输出为按区 chunk 的静态 JSON，运行时按需 fetch + 内存缓存；HTML/JS 与数据分别长缓存，改数据不失效代码缓存。构建校验闸门收缩为纯校验（枚举/契约/坐标全部拦截职责不变），注入职责随单文件架构一并退役。

**部署**：Wrangler 构建跑前端构建，产物 `dist/` 交静态资产服务；边缘 Worker 先匹配 `/api/*` 路由，再剥离 `/next-stop-gacha` 前缀交静态服务。Route 只接管 Lab 的游戏子路径，不绑定整个 hostname（Lab 首页仍由 Lab 仓库负责）。后端配请求级测试（Node，覆盖前缀剥离与 API 行为）。

## 信息组织

| 数据 | 来源 | 更新频率 | 存储 | 用途 |
|------|------|---------|------|------|
| 目的地主数据（城市 + 联程线路） | 模型知识分区生成 + 枚举校验 | 手工低频 | data/ 六区 + routes JSON → 校验后发布为静态 chunk | 筛选/展示/随机/行程 |
| 补丁（坐标/住宿/当地交通/体力/海拔/难度/同行） | 同上，独立补丁文件 | 手工低频 | data/ 补丁 JSON | 行程规划/路书/筛选 |
| 用户状态（收藏/对比/行程/打卡/皮肤选择） | 用户操作 | 实时 | localStorage（**真相源**） | 跨会话保留 |
| 云端中转（短链 payload） | 用户主动分享 | 低频 | Workers KV（带 TTL，可再生） | 跨设备分享 |

| 插画资产 | 文生图工单 + 人工终审 | 低频 | 仓库 webp（懒加载） | 手绘视觉 |
| 天气 | Open-Meteo 免 key API | 实时 + 缓存 | localStorage 缓存（TTL 3h） | 详情/路书参考 |

## 决策机制

- **过滤**：按属性语义分两类（判据＝低档位是「主动偏好」还是「被容忍的约束」）
  - **偏好型 → 多选 OR**（选中取并集）：地区、季节、冷热、体力（躺平/正常/费腿/硬核）——想小众/想硬核是主动选择，不自动联动
  - **容忍型 → 天花板**（点一档＝自动含所有更省/更易达的低档，chip 填充式点亮）：花费、抵达难度——高容忍必然接受低，单调
  - 玩法标签为 AND（收窄）；天数为桶映射（2/3/4-5/7/10+ 与 days 数组求交）；关键词全字段模糊
  - 高海拔(>2500m) 为独立二元排除/警示开关，与体力四档正交
  - 实现：两类共用 `state[key]` Set 与同一条 `!set.has(d.value)` 过滤；差异只在 chip 点击——容忍型点第 t 档填充 {0..t}，`.has` 天然得到"≤天花板"
- **空池治理**（AND 收窄语义保留）：两道防线——①事前：每个未选 chip 实时标注"点下去还剩几个"（按各组真实点击语义模拟），0 的置灰但仍可点；②事后：结果为 0 时计算去掉哪个单一条件能救回最多，按救回数给一键定向放宽（保留其余条件），扭蛋空池同样给最优放宽+一键再扭；「全部清空」为兜底
- **筛选台呈现**：筛选行标签一律两字（地区/季节/天数/冷热/花费/抵达/体力/同行/玩法——「抵达难度」收为「抵达」）；手机窄屏筛选台默认收纳为一行「搜索 + 筛选入口（带已选计数徽章）+ 清空」，点开就地展开完整行组，「抽一个」不随收纳、恒定首屏可见；桌面端保持全展开。收纳只是呈现折叠，不改变任何过滤语义
- **界面动作分区**（同类同形、按归属安放；任何新入口先问归属再落位，不得往筛选台底部堆）：
  - **全局入口挂靠语境、不集中占行**（手机一屏内容珍贵，低频入口不配独立一行）：足迹地图与备份都是打卡/收藏数据的动作，挂在页头足迹统计胶囊处——胶囊常显（零打卡时显引导文案而非隐藏，否则地图入口随之消失）、胶囊本身即地图入口（统计即入口）、统计文案 nowrap（宁可整体换行不得在胶囊内折行）；📤 备份为与胶囊**等高**的小图标钮随行（视觉不得超过胶囊高度、不悬浮页角——绝对定位的角落按钮在本设计语言里是异物），两件作为整体换行不拆散。图标钮视觉小于 44px 时经外扩热区补足触达 ≥44px（注意 absolute 伪元素基准是 padding box，外扩量须扣除边框），aria-label 补全名，不随筛选台收纳、恒定可见
  - **皮肤入口归页脚 meta 区**：换肤是站点级外观设置，与页面任何数据无关，且属「设一次即忘」的最低频控件（有「随机」选项可设完永不再碰）——归页脚（免责行同区），不占首屏、不挤头部；头部不放与数据无关的入口
  - **筛选台内只放筛选**：三个二元开关（避开高海拔/只看收藏/隐藏去过的）以「偏好」chip 组呈现，与九个筛选组同一交互语言（点亮态、徽章计数）；开关的过滤语义与正交性不因呈现收编而改变
  - **台底＝结果集视图控制**：排序 + 命中数 + 清空筛选，仅此三件；「清空筛选」为筛选的唯一清空入口（空态「全部清空重来」为兜底例外），收纳条不设重复清空按钮
  - **收纳条「筛选」入口带展开折角**（▼，展开时翻转）：可展开这件事必须自带暗示且一眼可见，不靠用户试出来
  - **不设原生 confirm**：可轻易重建的状态操作（对比池覆盖一类）直接执行 + toast 反馈，不用确认弹窗打断心流；真不可逆的动作才可考虑确认（现无一处原生 confirm）。外来数据导入的合并确认（分享链接）是自定义面板、语义不同（异端数据并集进本地），不在此列
- **属性打标原则**：不强制单值——地方有几副面孔就打几档（多值数组），没有明显个性的**留空=通配**；不硬拍主观档位。抵达难度除外（地理事实，天然单值）——difficulty 口径=**整程复杂度**：自出发地完成该卡全程（线路卡含全线）的总换乘/接驳复杂度，城市卡与线路卡同口径；班次密度细则：直达班次 <5 对/日慎标「直达」
- **条件随机**：在当前过滤结果内均匀抽样；动画减速后揭晓
- **顺路排序**：以上海为起点的最近邻贪心，用户可手动覆盖
- **顺路彩蛋**：候选=未在行程的**城市记录**（线路卡排除——展开装入是行程单动作）。两类语义同池混排——度量同为 km、取小者排序，<200km 取前 3，点击插入环线最优位置而非追加末尾：
  - **廊道顺路**（「+绕 Nkm」）：插进现有环线（上海→各站→上海）某段的最小绕路增量（三边差）；仅当该段与插入后两条子段**全为陆路**才成立——纯几何三边差在飞机段上没有顺路语义（直飞大圆航线正下方的点增量≈0，会推出「上海⇄拉萨顺路苏州 +0km」式胡说）。**「非飞机」判据=段估算的 air 布尔**（廊道与落脚短驳同规）：「飞机+包车」（noair 站组合档）同样是飞行段、地面包车只是末程接驳，不得比 mode 字符串——组合档一旦漏网，含 noair 远站的行程会让近沪城市以 +绕0km 涌入。近出发地城市不设禁入规则：纯陆路短途行程（去南京顺路苏州）的近沪推荐是正当顺路语义；远途行程的进出大交通必为飞行段，守卫本身即保证不推近沪
  - **落脚顺游**（「距X约Nkm」）：候选贴着行程中某个**非上海**站点（直线 ≤150km 且该段短驳非飞机）时，插在该站旁——「到了成都顺便乐山」语义；度量与展示用距锚定站的距离（支线往返的三边差会虚大，用户关心的量是"离多远"）
  - 候选池尊重「避开高海拔」「隐藏去过的」两个开关（其余浏览筛选条件不约束行程建议）；展示下限 1km（不显示 0）；点击插入=展示所用的同一个最优位置（锚定站旁/廊道段内），永不追加末尾；全程直飞且各站周边无卡的行程没有彩蛋，是诚实答案
- **交通启发式**：按坐标球面距离分档（短途打车/自驾 → 中程高铁 → 长程飞机），西藏/新疆/海南/港澳倾向飞机+包车。三条覆盖规则，优先级从高到低：
  1. **显式交通**：`stop.leg.transport`（游轮/轮渡/火车枚举，build 校验）直接指定该段方式，优先于一切启发式（游轮 🚢≈26km/h）——`leg` 只承诺逐日文案与住宿、不承诺陆路，非公路段（如三峡游轮）必须显式声明；`tripLegs` 从「进入该站」的目标站 leg 读 transport
  2. **leg 陆路例外**：相邻两站同属一条完整启用的 leg 组（`legEligibleIndices`+同 rid 判定）且无显式 transport 时 `overland=true`，60km 以上一律「包车/自驾」不判飞机——招牌自驾/穿越线（G318/阿里南线/沙漠公路）两站再远也是陆路，飞机判定与 leg 叙事矛盾即「编造」；中途过夜由 `leg.stays` 承载，站间显示累计陆路里程/时长。只作用于站间，上海往返大交通不受影响
  3. **进出门户**：线路可选 `entry`/`exit`=门户城市 id（与首末停留站不同，如三亚进海口出），`tripGateway()` 仅在行程恰为某线路完整整组时生效，改写首末大交通端点坐标与标注——行程单/逐日速览/详情/预算四处同源；门户→首站、末站→门户的短驳由 leg 文案承载

  整组被破坏（改序/删站/改天数/半装）时 overland 与门户一并失效，回退通用距离启发式。渤海/黄海跨海修正按方向生效——胶东三市/大连与真跨海对岸 ≥500km 判飞，同侧陆路走廊不强制；**江浙沪自驾修正**：两端都在江浙沪（含上海）的段，160~400km 档也标「高铁/自驾」，海岛登船段不受影响；输出恒明示为估算
- **预算估算**：每站消费档位×天数得住宿餐饮区间，加大交通按距离粗算，输出人均区间
- **路书装配**：按每站所选天数匹配（或就近降配）该站行程方案，串接站间交通段，输出逐日视图 + 纯文本导出 + 打印样式
  - **线路视角逐日文案**：线路 stop 可带可选 `leg={route, stays, transport?}`——「这一段路怎么走、当晚睡哪」的线路视角文案，路书对该 stop 优先用 leg 而非城市独立游 plan（城市 plan 描述「住这座城放射游」，与线路语义不符；线路 stop 指向大区域城市卡时，城市方案被压档会产出错误的「按 N 天版改编」内容——leg 是把线路自身叙事带进路书的机制）
  - **整组判定**：leg 是写死前后站顺序的「线路上下文」文案，故同 `r` 的全部 stop 必须齐全、连续、同序、天数=默认才整组启用；用户上移/删站/顺路排序/只装半条 → 整组失效回退城市 pickPlan；trip 条目记 `r=线路 id` 供路书回查
  - **旧持久化降级**：loadLS 遇失效 `r`（非现存线路 id）去标记转独立城市站、天数夹到该城市合法档——线路 id 已丢无法安全重建，明确降级不假称已修
  - 缺 leg 的线路 stop（途经点是真目的地城市）走城市 plan 就近降配
  - **逐日骨架**：路书开头给逐日速览，每天一行「日期/D 序号 · 当日动作（赶路段=从 A 到 B+交通方式，停留日=游 X）· 宿 X」；宿地来源：基地型城市=城市名，路线型=方案级 `stays` 每晚落脚点数组（build 校验长度==该方案 days）；行程单可选出发日期（不填维持 D1/D2 记法）；速览是新增层，逐站详情段（方案/美食/住宿/市内/天气）全保留
- **分享/备份（无后端兜底通道）**：打卡+收藏的跨设备迁移，三形态同源：①分享链接=URL hash 携带 favs/visited id 列表，打开时弹确认后**并集合并**进本地（绝不覆盖行程/对比，导入后清 hash）；②二维码=零依赖 QR 编码器（byte 模式+纠错 M，canvas 绘制），超长（>约 1500 字符）提示改用 JSON；③JSON 导出/导入。入口克制：工具行小按钮开小弹层。后端短链/同步码（见「后端」）是其增强形态，合并语义同源
- **卡片收录原则**：①城市卡之间只能并列，包含关系只允许线路卡→城市卡（stops 引用）；②卡名=大众目的地语义而非行政区划——纯名义行政包含不改名不算违例；③城市卡=单城市级别，环线/多点串联玩法归线路卡；④一张卡的行程方案不得吃掉另一张独立卡的核心内容——越界日程改域内内容+「可接《某卡》」提示；⑤同名歧义收窄命名消解（黄山→黄山风景区）
- **实时天气**：唯一的外部实时接入，Open-Meteo 免 key + localStorage 缓存（TTL 3h、超 60 条淘汰最旧）+ AbortController 超时 + 任何失败静默降级——外部接口永远不是页面可用性的前提。异步填充防串数据（详情 data-id 核对、路书世代计数）。署名合规：CC BY 4.0——详情天气卡与路书免责行给出来源+许可链接+「经本站整理换算」，文本导出携带同款署名。**部署约束：Open-Meteo 免费端点仅限非商业使用**，本站为个人免费项目符合前提；若商业化必须换源
- **玩法节奏（基地型 vs 路线型）**：绝大多数目的地是基地型（住一处放射游）；少数记录玩法主体是沿线移动、按晚换落脚点。路线型用小清单标注（不进数据 schema、不做筛选项），路书对应站在住宿行前给「沿线多点换宿」提示。收录口径从严：单基地+一晚特色住宿与双基地不算；吞并其他独立卡的环线不允许以城市卡形态存在，现存路线型城市卡的途经点均无独立卡、暂维持，若后续为其途经点建卡须同步迁移。线路卡无需此标注——装入行程即展开为多站
- **内容质检规程**：`.agent/content-checklist.md` 为**常驻活文件**（非消费型工单）——存量体检（M48/M56 一类）、新卡入库自检（扩容批/M22 北京批/M11 海外）、跨家族 review 对内容批的抽查，三场景共用同一清单；内容模块开工先读、收工回写本轮沉淀的新规（注模块号）；清单与本文件冲突以本文件为准，规则升级为机制的由 cc 收编进本文件。红线总纲=宁缺毋编

## 主题皮肤系统

视觉不是单一风格，而是**可切换的皮肤库**。一套 DOM、多套皮肤：皮肤 = CSS token 集 + 字体对 + 固定小件插画 + 装饰件开关；切换只换表达，不换结构与交互。

**机制**：
- **新皮肤诞生流程（画风基准约定）**：每个皮肤以 **codex 整页 mock** 起点——用户认可的 mock 即该皮肤**唯一画风基准**：存 `raw/<id>/style-ref-mock.png`、挑版时转 `picked/<id>/` 永久入库；风格锁按 mock 校准预置（工单），固定小件批每张生成都以 mock 为 image reference。想做新皮肤=先让 codex 画一张整页 mock 给用户看。mock 只锚风格、不锚角色（角色参照见吉祥物条）；目的地共享集不在此列（皮肤无关，基准=picked/dest/ 三样张）
- **token 纪律**：所有可换肤样式（色板、底纹、描边宽、投影、圆角族、字体对）必须经 CSS 自定义属性表达，视图模块 CSS 只引用 token；根元素 `data-theme` 切换 token 集。功能语义色（行程橙/对比蓝/打卡绿等 UX 语义、九区色）各皮肤可重映射色值但语义分工不变
- **皮肤声明**：每皮肤一份声明（token 文件 + 字体对 + 资产目录 + 装饰件开关位）；新增皮肤=新增声明，不改既有皮肤。token 文件以构建期 scoped block（`:root[data-theme=<id>]`）参与打包，切换=换 token 作用域，不做运行时样式装载；**资产目录/装饰开关**由渲染路径按当前皮肤声明真实消费。**字体对不走这条路**：运行时字体选择完全由 scoped CSS token（`--round`/`--sans`）驱动——浏览器只在某个 `font-family` 真的被用来画字时才取 `@font-face` 的 `src`，非当前皮肤天然不发起该字体的请求，这就是懒加载/失败回退的完整机制，不需要也没有任何 JS 读取声明里的 `fonts` 字段去生成 `<link>`/`FontFace`。声明里的 `fonts` 字段是**静态元数据**，唯一用途是被测试拿去核对与 CSS 里 `@font-face`/`--round`/`--sans` 的字面量是否一致（防止两处各写一份却悄悄分叉的 drift-pin），不是运行时输入
- **字体属皮肤维度**：每皮肤 ≤2 个字体（标题+正文），构建期 pyftsubset 子集化（对 data 全量文本 + UI 文案算字符集，单字重 ≤1MB woff2）；非当前皮肤的字体懒加载；字体失败回退系统字体，排版不塌。**子集语料随产物落盘进 git**（`*.corpus.txt`）——woff2 是二进制黑盒，语料文本是它唯一可断言的影子；font-corpus 测试断言「源文件/数据字符 ⊆ 语料」，新增文案或数据没重跑子集化时机器红灯，而不是上线后现系统字体回退（子集化管线是本地手动工具，天然有「改了字忘了跑」缺口）
- **选择与持久化**：设置入口选皮肤，含「随机」选项（每次进站随机一套，呼应扭蛋气质）；localStorage 记忆；默认皮肤首屏直出不闪烁
- **成本模型（皮肤边际成本三层）**：①UI chrome=纯 token/CSS，便宜；②固定小件插画（扭蛋机/吉祥物/空态/九区题头，约 14 张）+ 工艺件（纹理/印章/垫底，2~6 张——边框/分隔线槽位已拍板撤出成套清单，隐形槽位实证见「工艺件画布契约」）=每皮肤一批文生图，中等；③**目的地插画不随皮肤换**——张数×皮肤数线性爆炸不可行，目的地图统一以「贴进手帐的照片/明信片」语义嵌白框（风格隔离舱），全皮肤共用一套；某皮肤要成套目的地图属独立铺量决策
- **共享集皮肤适配滤镜**（白框隔离舱的第二道调和层，零资产成本）：每皮肤可在声明中带一条作用于目的地图的 CSS 滤镜链（token 表达，运行时渲染、不生成文件、未来批次自动继承）——青花=灰度→钴蓝双色映射（inline SVG feComponentTransfer 把明度斜坡映到瓷白~钴蓝，`filter: url(#…)`）；doodle=灰度线稿化（grayscale+对比提升）；山水=轻度降饱和融纸；奶油/手帐=原色。局限诚实记录：色调映射改的是色不是笔触（出不来缠枝莲式纹样），白框「贴画」语义下够用；追求本体笔触仍走成套铺量条款。若滤镜后的卡图网格出现渲染性能问题，退路=M42 管线构建期预生成着色变体（拿字节换渲染）
- **吉祥物跨皮肤同一角色**：咔啦在每个皮肤有该皮肤笔法的版本，**角色形象以通过版为跨皮肤角色参照（皮肤 mock 只锚定风格、不锚定角色）**；辨识三件套（渔夫帽/背包/纸地图）形状恒定，颜色随皮肤语言（默认黄帽红包，单色皮肤如青花随色系）；IP 红线不变；名字用于文案人格

## 后端（边缘 API）

**总纲**：Cloudflare Workers 免费计划 + KV 免费配额内运行，**绝不绑付费**；后端只做锦上添花（分享/同步），任何超量/故障时 API 静默降级（同天气哲学），核心功能零依赖后端。localStorage 恒为用户数据真相源，云端只是中转。

- **短链分享**：`POST /api/share` 收 `{type: trip|marks, payload}`——payload 大小上限（≤8KB）+ schema 校验 → 生成 6 位短码 → KV put（write-once，TTL 180 天）；`GET` 短码路由返回 payload，前端渲染只读路书/导入确认。滥用面：写侧按 IP 简单限流、读侧天然只读。短码空间碰撞时重掷
- **同步码云同步**：随机 12 位数字码即 Durable Object 实例名（码即凭证，无账号无密码，页面明示「码不外泄」，严格 POST-only——PUT 只能覆盖已由 POST 建过档的码，过期/未知码不可复活）；**PUT 在服务器端做并集合并**（读现值→与请求 payload 取并集→写回，而非信任客户端整份覆盖），GET 拉取后前端再走一次既有**并集合并**语义（与链接/JSON 导入同源，绝不覆盖行程/对比）；行程不同步（行程是短命工作台，打卡/收藏才是长命资产）。存储选型是 Durable Object 而非 KV：plain KV 没有跨请求原子 read-modify-write、同一 key 每秒限写 1 次，两台设备真并发同步会丢数据；DO 的 input/output gate 保证同一实例（同一个码）上的并发调用被串行化，真正原子。TTL 靠 DO 自身 alarm 机制实现（每次 POST/PUT 续期，到点自毁存储），不是 KV 的 `expirationTtl`
- **限流**：按 IP 简单防滥用（短链创建/同步码创建/同步读写三个独立配额），同样建在 Durable Object 上而非 KV 计数器——KV 计数器在真实同 key 写频限制下会被绕过或直接故障，DO 每个 (bucket, ip, day) 一个实例，串行 read-modify-write 不会跑偏
- **API 与静态资产同 Worker**：`/api/*` 路由先于前缀剥离匹配；请求级测试覆盖短码读写、限流、超限降级、损坏 payload 拒绝、真并发下的 DO 原子性（不能只测顺序请求）

## 存储与精确性

- 必须精确：schema 枚举字段（构建时强校验，错即拒绝发布）
- 允许漂移：坐标（两位小数，仅用于估算）、住宿/交通建议（知识性文案，页面挂免责）
- 可再生：KV 中的短链 payload、Durable Object 中的同步快照与限流计数——丢失可接受，本地真相源随时可重新上传
- 用户输入：无必填输入；同步码/短码为仅有的用户可粘贴凭证，输入侧做格式校验

### 线路卡数据契约（routes.json）

- 线路记录与城市记录同构（全字段必填，直接复用既有校验与筛选），另有：`regions[]`（途经大区，多选 OR；`region` 保留为主大区徽章）、`stops[]`、可选 `entry`/`exit`（进出门户城市 id）。**`stops` 的存在与否是线路卡的唯一判别标志**（不新增 type 字段）
- `stops`：`{id, days, leg?}[]`，长度 2~4，id 必须存在于城市池、不得重复；stop.days ∈ [1, 该城市 days 上限]；`leg={route, stays[], transport?}`，stays 长度==stop.days，transport ∈ {游轮, 轮渡, 火车}
- 天数契约：站点天数分配是唯一事实真相=默认装入分配；`days` 枚举是有对应 plan 文字支撑的弹性档位，须满足 min(days) ≤ Σstop.days ≤ max(days)，且每档 ∈ [站数, Σ城市上限] 可达；线路名称不带日数（构建器强制），卡面展示「约min~max天 · 默认Σ天」；装入行程后站点天数放开 1~城市上限
- 海拔保守传播：任一停留站 alt=true 则线路必须 alt=true；build 另有人工海拔断言硬钉子（ALT_TRUE_PIN）与「文本含达坂/垭口/3000m+ 但 alt:false」的非阻塞审计警告
- plan 文本站点顺序与 stops 逆行时给非阻塞警告（漏站判不了，靠评审）
- 路线型城市记录：plans[].stays 可选（每晚落脚点），长度==该方案 days
