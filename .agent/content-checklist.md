# 内容体检清单（城市卡/线路卡质检规程）

> 性质：**常驻活文件**，不同于消费即归档的工单。三类场景共用：①存量体检（M48 线路体检、M56 通达性体检等）；②新卡入库前自检（扩容批 / M22 北京批 / M11 海外版）；③跨家族 review 对内容批的抽查。
> 用法：内容模块**开工先读本清单，收工把本轮沉淀的可复用新规回写进来（注模块号）**，并在 state 收工条目提一句。清单与 design 冲突时以 design 为准并回报修正。
> 红线总纲：**宁缺毋编**——交通/住宿/玩法文案必须真实存在，查不到实证就改写为不承诺的表述。
> 姊妹规程：插画轨道=`.agent/illustration-brief.md`（皮肤资产成套清单与批工单）。

## 一、通用（城市卡 + 线路卡）

- [ ] schema 必填字段齐全，`python3 tools/build.py` 过闸——枚举字段错即拒发；精确性分层按 design「存储与精确性」：枚举必须精确，坐标/住宿交通建议允许知识性漂移（页面挂免责）
- [ ] 收录原则五条（design「卡片收录原则」，M31）：城市卡间只并列；卡名=大众目的地语义而非行政区划；城市卡=单城级别，环线归线路卡；行程方案不吃掉别卡核心内容（越界改「可接《某卡》」提示）；同名歧义收窄命名
- [ ] 属性打标（design「属性打标原则」）：多面孔打多档、无明显个性留空=通配、不硬拍主观档位；抵达难度例外（地理事实单值）
- [ ] 高海拔：>2500m 必须 `alt:true`；线路海拔保守传播（任一停留站 alt 则线路 alt）；「文本含达坂/垭口/3000m+ 但 alt:false」的构建审计警告不得靠无视过关
- [ ] 住宿文案去品牌化（M12 口径）：不点名酒店品牌，写档位与区位
- [ ] 出发地视角：transit/difficulty 现全量为**上海视角**（M22 参数化落地前，新卡一律按此口径写）
- [ ] transit 文案：时长口径一律带「约」（估算语义），不承诺具体班次/时刻；所述交通方式与该城通达性（见二节）不得矛盾
- [ ] 玩法节奏口径（design「玩法节奏」）：路线型（按晚换宿）从严认定，单基地+一晚特色住宿不算；吞并独立卡的环线不得以城市卡形态存在
- [ ] 特色主题类候选「值得住」判定：产业/主题看点须叠加**第二支点**（景观/古镇/温泉/度假底盘）才过关——纯市场型（批发/电商集群）、半日研学园、单体主题公园/幻城类从严或交拍板；强季节主题（冬捕/菌季/龙虾季/发射窗口）须在 seasonNote 写明窗口月份与淡季落差（M49）
- [ ] 调研落选记录与候选同等归档（含「并入既有卡/补 tag」建议），避免重复调研；新 tag 提案须附可回标数量级估算，未达约 10 张门槛的标「条件推/暂缓」（M49）
- [ ] tag=筛选契约，**一个 tag 一种出行动机**——不得为凑回标数把不同动机合并成一个 tag（M49 教训：「星空航天」混装观星与追火箭两种动机被用户打回，拆为 `观星` 单立、航天由卡内容承载）；量不够的动机宁可不立 tag、靠文案+搜索承载（M49 二拍）

## 二、通达性与交通表意（M56 起新增；网查严查，逐项留依据）

### 判定与标记（noair / norail / slowrail 三守卫字段）

- [ ] **民航**：目的地有无**现役**民用机场与现存客运航线（新开/停航都要查最新状态，不凭印象）；无 → 标 `noair:true`
- [ ] **轨道**：有无铁路客运站与常态客运班次；高铁/普速分清——transit 文案不得把普速区间写成「高铁」；无客运轨道 → 标 `norail:true`
- [ ] **仅普速**（M56 追加，2026-07-21）：有轨道但**无高铁/动车常态班次** → 标 `slowrail:true`（附一句依据，与 norail 互斥）——挡启发式 leg 的「高铁」编造档（漠河类：直线 <950km 会被标「高铁 约4.8h」而现实是 K 夜车 13h）；该城动车化/通高铁后翻转标记（挂高时效复卡清单）
- [ ] noair/norail 判定按**本体粒度三档口径**（M56 拍板，全文见 design M56）：①本体或 <30km 同名近场→通达；②同市异名门户 ≤1h 且 ≥4班/日→rail 通达（文案写明转乘）、air 仍按本体；③远端门户→按本体标；品牌门户机场（以目的地命名、50km 级）与多市共享机场不标 noair
- [ ] noair 与 norail **独立判、逐项查、禁止连坐**；「偏远=无机场」直觉必翻车——凡判 noair 前必须逐城搜「<县名>机场」（M56 教训：普兰/五台山/安阳三例均栽在被忽略的既有低频支线机场）
- [ ] 通用机场（A1/短途运输）、直升机/低空经济航线、军用机场均**不计入民航客运**；「有机场」三字不构成豁免（M56：自贡凤鸣/嵊泗直升机/崇明兴城军用）
- [ ] 机场「停航改扩建/在建校飞/刚复航」是独立状态，不硬置 noair、挂「复核提醒」清单定期回查（M56：泸沽湖/冠豸山/牡丹江停航，翼云/蒙自/元阳在建，天水/丹东刚复航）

### 查证纪律

- [ ] 查证顺序：官方/权威渠道优先（航班动态、12306 可查到站、地方文旅官宣），旅行攻略只作旁证；**每个 noair/norail 标记附一句依据**，清单交用户过目后才落库
- [ ] 一致性互查：transit 写「飞 XX 转包车」的城市 noair 标记要对得上；写「高铁直达」的必须真有班次；seasonNote/plan 里的交通叙述与标记不得互相矛盾
- [ ] 新线双向时效：新开高铁/机场未回写（雁门关/应县/松阳/黟县/日喀则）与保守文案过时（宜兴沪苏湖后已直达）都要查——内容批对近 2 年新线沿途卡强制复查（M56）

### transit 文案表意

- [ ] **自驾/包车习惯区**：北疆/伊犁、青甘环线、川西、藏区、内蒙草原深处等成熟包车/自驾走廊，站间与进出段表意须符合当地主流玩法——**按习惯列自驾/包车主选项，确实有航线的并列飞机（组合标签），通不了的绝不列**
- [ ] 旅游专列/季节性普速列车（Y 字头等）不得表述为「动车/高铁」——动车组**车底**≠动车**班次**；支线终点站须核实是否真在 D/C/G 停靠序列内（M48；茶卡教训）
- [ ] 高铁时长写实测直达区间勿取理想值：与主流时刻表口径差 >1h 即失真候选；直飞时长按实际飞行取半小时粒度、勿系统性取低（M48；太原/洛阳教训）
- [ ] 承诺性易变信息（船期/新开航线/景区开放/边境政策）用「以当期实查为准」对冲（M48 正例：三峡游轮/齐鲁/海南西线）
- [ ] 本县/区 2020 年后新通航支线机场须在 transit 体现（注明班次稀/高高原天气限制）；「（旺季）直飞X机场」必须锚定**当期**复航城市+年份，禁止历史通航列表兜底（M56：塔县/巴音布鲁克/九黄教训）
- [ ] 轨道等级三分：「高铁到拉萨」类永远为假可作硬规则；设计时速 ≤200km 的 D 字头线路写「动车」不写「高铁」（宜万/温福/宁启/牡绥）；同一条线全库措辞统一（M56）
- [ ] 「转车」措辞：目的地本体无客运铁路时写明「转长途汽车/包车」，不得用「转车」暗示铁路换乘（M56：蔚县/易县教训）
- [ ] 站名硬伤排查：非官方站名（「天目湖站」实为公交站）、已更名站（永嘉→温州北）、普速站与高铁站同名异站（天柱山站≠潜山站；京原五台山站≠雄忻在建站）——新卡入库时站名必须对 12306 实名（M56）

## 三、线路卡专项（M48 体检项同源，design「线路卡数据契约」）

- [ ] 站序方向符合真实走向；plan 文本站点顺序与 stops 逆行会有非阻塞警告，漏站靠人审补位
- [ ] leg 完整性：同线全部 stop 的 leg 齐全成组；`stays` 长度==stop.days（build 强校验）；leg 文案写死的前后站顺序与 stops 一致
- [ ] 线路 leg：招牌自驾/穿越线的相邻站必须靠 leg 整组 overland 语义承载（不得靠距离启发式碰运气）；非公路段（游轮/轮渡/火车）必须显式 `leg.transport` 声明（原列二节，2026-07-21 规整归位本节）
- [ ] 天数契约：min(days) ≤ Σstop.days ≤ max(days)、每档可达、线路名不带日数（build 强制）
- [ ] 门户进出：entry/exit 与叙事一致（三亚进海口出一类），门户城市 id 真实存在于城市池
- [ ] transit 真实性（编造红线）：首末大交通与站间描述均须真实可行——本节与二节联查
- [ ] plan 文本提及的地名与 stops/域内内容一致，不越界吃掉独立卡内容
- [ ] 异地进出型（进A出B非环线）线路：transit 宜自述**两端**门户可达性，出口不能只靠 local 隐含（M48；正例 route-tarim-desert「线路在库车结束」）
- [ ] 线路卡 transit 时长与同名停留城市卡口径交叉自洽（同一城两处时长不一致即失真候选）；difficulty 交叉核对同理，全库单一口径已拍定=**整程复杂度**（自出发地完成该卡全程的总换乘/接驳复杂度，城市卡与线路卡同口径），班次密度细则：直达班次 <5 对/日慎标「直达」（M48 拍板）
- [ ] 线路日程实体在城市池另有独立卡 → 优先「可接《X卡》」指引而非整段并入；确属线路招牌主题的有意重叠须显式登记豁免（收录原则④执行细则，可按独立卡名单机械预筛）（M48）
- [ ] 无 leg 对象的多城线：火车/轮渡段以 transit/local 散文披露即可，leg.transport 显式声明规则仅约束存在 leg 的线路（M48 现状 codify）

## 四、季节与时间文案（M55 起新增）

- [ ] seasonNote 时间语义尽量带月份数字（「桃花季（3月底-4月初）」优于裸「桃花节期间」）——路书当季过滤按月份解析，纯节庆名解析不出月份会造成语境错配
- [ ] seasons 数组与 seasonNote 叙事一致（文案主推冬景而 seasons 无「冬」= 矛盾）
- [ ] M55 落地后配套离线审计会产出「含时间语义但解析不出月份」句子清单，按清单逐句补写月份

## 五、流程

- **存量体检**：只读产出发现清单走 F 流程 triage，**不直接改数据**（M48 纪律）
- **体检编排范式**（M48 首跑沉淀）：机械项先离线脚本清零 → 判断项按地理走廊分片派子代理网查 → 驱动改动的摘录逐条与 data 原文核对（防幻觉摘录）→ 事实性阻塞/警告全部走换视角反方复核（只验依据链）→ 分级工单交 triage。plan 漏站类机械提示存在分词误报（站名简称/景点名/辖区名），一律以人审定夺为准
- **通达性体检补充范式**（M56 沉淀）：并行大批分片时注意会话级搜索配额共享（WebSearch 200/会话）——后到分片留「存疑」不硬拍，配额耗尽改道 WebFetch 维基/聚合或 tavily；所有「存疑」项收口时统一补查定案；「高时效复卡清单」（在建/停航/新线）随 findings 挂 state 定期回查
- **新卡入库**：过本清单自检 → build 过闸 → 新增批由另一族模型 review 时按本清单抽查
- **回写纪律**：每个内容模块收工时把新规则追加进对应节（注模块号）；规则升级为机制的（如通达性守卫字段）由 cc 收编进 design，本清单保留执行细则
