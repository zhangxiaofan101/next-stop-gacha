# 内容体检清单（城市卡/线路卡质检规程）

> 性质：**常驻活文件**，不同于消费即归档的工单。三类场景共用：①存量体检（M48 线路体检、M56 通达性体检等）；②新卡入库前自检（扩容批 / M22 北京批 / M11 海外版）；③跨家族 review 对内容批的抽查。
> 用法：内容模块**开工先读本清单，收工把本轮沉淀的可复用新规回写进来（注模块号）**，并在 state 收工条目提一句。清单与 design 冲突时以 design 为准并回报修正。
> 红线总纲：**宁缺毋编**——交通/住宿/玩法文案必须真实存在，查不到实证就改写为不承诺的表述。

## 一、通用（城市卡 + 线路卡）

- [ ] schema 必填字段齐全，`python3 tools/build.py` 过闸——枚举字段错即拒发；精确性分层按 design「存储与精确性」：枚举必须精确，坐标/住宿交通建议允许知识性漂移（页面挂免责）
- [ ] 收录原则五条（design「卡片收录原则」，M31）：城市卡间只并列；卡名=大众目的地语义而非行政区划；城市卡=单城级别，环线归线路卡；行程方案不吃掉别卡核心内容（越界改「可接《某卡》」提示）；同名歧义收窄命名
- [ ] 属性打标（design「属性打标原则」）：多面孔打多档、无明显个性留空=通配、不硬拍主观档位；抵达难度例外（地理事实单值）
- [ ] 高海拔：>2500m 必须 `alt:true`；线路海拔保守传播（任一停留站 alt 则线路 alt）；「文本含达坂/垭口/3000m+ 但 alt:false」的构建审计警告不得靠无视过关
- [ ] 住宿文案去品牌化（M12 口径）：不点名酒店品牌，写档位与区位
- [ ] 出发地视角：transit/difficulty 现全量为**上海视角**（M22 参数化落地前，新卡一律按此口径写）
- [ ] 玩法节奏口径（design「玩法节奏」）：路线型（按晚换宿）从严认定，单基地+一晚特色住宿不算；吞并独立卡的环线不得以城市卡形态存在

## 二、通达性与交通表意（M56 起新增；网查严查，逐项留依据）

- [ ] **民航**：目的地有无**现役**民用机场与现存客运航线（新开/停航都要查最新状态，不凭印象）；无 → 标 `noair:true`
- [ ] **轨道**：有无铁路客运站与常态客运班次；高铁/普速分清——transit 文案不得把普速区间写成「高铁」；无客运轨道 → 标 `norail:true`
- [ ] **自驾/包车习惯区**：北疆/伊犁、青甘环线、川西、藏区、内蒙草原深处等成熟包车/自驾走廊，站间与进出段表意须符合当地主流玩法——**按习惯列自驾/包车主选项，确实有航线的并列飞机（组合标签），通不了的绝不列**
- [ ] 查证顺序：官方/权威渠道优先（航班动态、12306 可查到站、地方文旅官宣），旅行攻略只作旁证；**每个 noair/norail 标记附一句依据**，清单交用户过目后才落库
- [ ] 一致性互查：transit 写「飞 XX 转包车」的城市 noair 标记要对得上；写「高铁直达」的必须真有班次；seasonNote/plan 里的交通叙述与标记不得互相矛盾
- [ ] 线路 leg：招牌自驾/穿越线的相邻站必须靠 leg 整组 overland 语义承载（不得靠距离启发式碰运气）；非公路段（游轮/轮渡/火车）必须显式 `leg.transport` 声明

## 三、线路卡专项（M48 体检项同源，design「线路卡数据契约」）

- [ ] 站序方向符合真实走向；plan 文本站点顺序与 stops 逆行会有非阻塞警告，漏站靠人审补位
- [ ] leg 完整性：同线全部 stop 的 leg 齐全成组；`stays` 长度==stop.days（build 强校验）；leg 文案写死的前后站顺序与 stops 一致
- [ ] 天数契约：min(days) ≤ Σstop.days ≤ max(days)、每档可达、线路名不带日数（build 强制）
- [ ] 门户进出：entry/exit 与叙事一致（三亚进海口出一类），门户城市 id 真实存在于城市池
- [ ] transit 真实性（编造红线）：首末大交通与站间描述均须真实可行——本节与二节联查
- [ ] plan 文本提及的地名与 stops/域内内容一致，不越界吃掉独立卡内容

## 四、季节与时间文案（M55 起新增）

- [ ] seasonNote 时间语义尽量带月份数字（「桃花季（3月底-4月初）」优于裸「桃花节期间」）——路书当季过滤按月份解析，纯节庆名解析不出月份会造成语境错配
- [ ] seasons 数组与 seasonNote 叙事一致（文案主推冬景而 seasons 无「冬」= 矛盾）
- [ ] M55 落地后配套离线审计会产出「含时间语义但解析不出月份」句子清单，按清单逐句补写月份

## 五、流程

- **存量体检**：只读产出发现清单走 F 流程 triage，**不直接改数据**（M48 纪律）
- **新卡入库**：过本清单自检 → build 过闸 → 新增批由另一族模型 review 时按本清单抽查
- **回写纪律**：每个内容模块收工时把新规则追加进对应节（注模块号）；规则升级为机制的（如通达性守卫字段）由 cc 收编进 design，本清单保留执行细则
