# Goal — 下一站，去哪玩（Next Stop Gacha）

> 本文件由用户拥有。以下内容取自用户在 2026-07-12 会话中的原话与确认，如有出入以用户修订为准。

## 电梯陈述

给选择困难症用户做一个"好玩的"旅行目的地选择器网页：把中国值得住几天的城市/小地方（不只旅游城市，也要小众地）尽可能多地收进来，既能按条件筛选比对，也能设好限制条件后直接随机——避免选择困难。先做中国版。

## 用户画像

- 常驻江浙沪，短途默认江浙沪一带；长途默认从上海出发往返
- 住宿固定偏好高档连锁酒店档位（具体品牌偏好不在公开文案中出现——2026-07-13 用户要求产品文案去品牌化，偏好细节存 assistant 私有记忆）
- 自认选择困难，喜欢"随机决定"类交互
- 审美：浅色卡通画风（明确否决过深色炫酷风）

## 核心问题（为什么做）

1. **信息聚合**：一个地方值不值得去、什么季节去、玩几天、吃什么看什么，散落各处 → 每个目的地带结构化属性（美食/博物馆/古建/特色/多套行程方案）
2. **选择困难**：条件内随机（扭蛋），而不是无限刷列表
3. **从"选地方"到"能出发"**：多目的地串成完整行程 → 自动排顺路顺序、推荐站间交通方式、按偏好推荐住宿、生成可带走的路书

## 硬约束（已定）

- 单文件静态网页，零依赖、无后端、无登录；发布为 Claude Artifact（CSP 禁外联，字体/图片/API 均需自包含）
- 浅色卡通画风
- 数据为人工知识生成 + schema 校验，不接实时 API；页面上明示"以 12306/酒店 App 实查为准"

## 明确不做（Out of scope）

- 实时数据（天气、房价、车次余票）
- 用户系统 / 云端同步（本地 localStorage 即可）
- 地图瓦片（CSP 禁外联）

## 长期方向

- 海外版（用户原话："先做个中国版的"，暗示后续可能有海外版）
- 去过打卡 / 足迹统计
