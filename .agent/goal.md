# Goal — 下一站，去哪玩（Next Stop Gacha）

> 本文件由用户拥有。以下内容取自用户在 2026-07-12 会话中的原话与确认，如有出入以用户修订为准。

## 电梯陈述

给选择困难症用户做一个"好玩的"旅行目的地选择器网页：把中国值得住几天的城市/小地方（不只旅游城市，也要小众地）尽可能多地收进来，既能按条件筛选比对，也能设好限制条件后直接随机——避免选择困难。先做中国版。

## 用户画像

- 常驻江浙沪，短途默认江浙沪一带；长途默认从上海出发往返
- 住宿固定偏好高档连锁酒店档位（具体品牌偏好不在公开文案中出现——2026-07-13 用户要求产品文案去品牌化，偏好细节存 assistant 私有记忆）
- 自认选择困难，喜欢"随机决定"类交互
- 审美：浅色卡通画风（明确否决过深色炫酷风）

## 核心问题（为什么做）

1. **信息聚合**：一个地方值不值得去、什么季节去、玩几天、吃什么看什么，散落各处 → 每个目的地带结构化属性（美食/博物馆/古建/特色/多套行程方案）
2. **选择困难**：条件内随机（扭蛋），而不是无限刷列表
3. **从"选地方"到"能出发"**：多目的地串成完整行程 → 自动排顺路顺序、推荐站间交通方式、按偏好推荐住宿、生成可带走的路书

## 硬约束（已定）

- 单文件静态网页，零依赖、无后端、无登录；唯一内容源保持为本仓库，正式入口迁至 Cloudflare Workers 路径 `https://lab.medspiral.com/next-stop-gacha/`，每次 `main` 更新自动发布；迁移验收前 GitHub Pages 仅作回滚源，验收后取消旧入口发布；保持可打包回 Claude Artifact 的自包含性
- 浅色卡通画风
- 核心数据为人工知识生成 + schema 校验；页面上明示"以 12306/酒店 App 实查为准"。实时接入（2026-07-15 修订）：仅限免 key 的开放 API 纯前端调用（如天气预报），必须断网/接口失效优雅降级，不得成为页面可用性的前提

## 明确不做（Out of scope）

- 房价 / 车次余票实时数据（2026-07-15 复评维持不做，理由更正：无免费开放数据源，需自建后端 + 商业接口，与零依赖纯静态架构冲突；天气自此移出不做清单，见硬约束修订）
- 用户系统 / 云端同步（本地 localStorage 即可）
- 第三方瓦片地图（Leaflet + 高德/天地图 等，2026-07-15 复评维持不做：破坏单文件零依赖，且 GCJ-02 瓦片与本项目 WGS 近似坐标存在偏移；零依赖内嵌 SVG 中国地图不在此列，与打卡足迹合并排期）
