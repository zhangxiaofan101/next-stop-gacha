# Goal — 下一站，去哪玩（Next Stop Gacha）

> 本文件由用户拥有。以下内容取自用户在 2026-07-12 会话中的原话与确认，如有出入以用户修订为准。

## 电梯陈述

给选择困难症用户做一个"好玩的"旅行目的地选择器网页：把中国值得住几天的城市/小地方（不只旅游城市，也要小众地）尽可能多地收进来，既能按条件筛选比对，也能设好限制条件后直接随机——避免选择困难。先做中国版。

## 用户画像

- 常驻江浙沪，短途默认江浙沪一带；长途默认从上海出发往返
- 住宿固定偏好高档连锁酒店档位（具体品牌偏好不在公开文案中出现——2026-07-13 用户要求产品文案去品牌化，偏好细节存 assistant 私有记忆）
- 自认选择困难，喜欢"随机决定"类交互
- 审美：浅色手绘画风（2026-07-17 自「浅色卡通」演进：手写字体+手绘质感+手绘插画；深色炫酷风维持否决）

## 核心问题（为什么做）

1. **信息聚合**：一个地方值不值得去、什么季节去、玩几天、吃什么看什么，散落各处 → 每个目的地带结构化属性（美食/博物馆/古建/特色/多套行程方案）
2. **选择困难**：条件内随机（扭蛋），而不是无限刷列表
3. **从"选地方"到"能出发"**：多目的地串成完整行程 → 自动排顺路顺序、推荐站间交通方式、按偏好推荐住宿、生成可带走的路书

## 硬约束（已定）

- **架构（2026-07-17 修订，取代原「单文件零依赖无后端」约束）**：前端正规工程化（构建产物仍为纯静态资产）；允许一个简单后端，但仅限 Cloudflare 免费额度（Workers + KV 一类），**绝不进入付费档**——超量/故障时 API 静默降级，核心功能（筛选/扭蛋/行程/路书/地图）必须在后端完全不可用时照常工作。唯一内容源保持为本仓库，正式入口 `https://lab.medspiral.com/next-stop-gacha/`，每次 `main` 更新自动发布。原「可打包回 Claude Artifact 的自包含性」要求随单文件约束一并退役
- **浅色手绘画风（2026-07-17 修订）**：手写字体 + 手绘质感 + 手绘插画，仍为浅色暖调。目的地全量插画排期靠后；扭蛋机等核心元素的手绘化排期靠前；插画样例先行、用户终审通过后才铺量
- 无登录：不做账号体系；云同步以匿名同步码实现
- 核心数据为人工知识生成 + schema 校验；页面上明示"以 12306/酒店 App 实查为准"。实时接入：默认仍为免 key 开放 API 纯前端调用（如天气预报），必须断网/接口失效优雅降级，不得成为页面可用性的前提；将来若需带 key 数据源，须走自有 Worker 代理并另行拍板

## 明确不做（Out of scope）

- 房价 / 车次余票实时数据（无免费开放数据源，需商业接口，与「绝不付费」约束冲突；2026-07-15 结论维持）
- 真实用户账号体系（注册/登录/密码找回）——原「用户系统/云端同步不做」于 2026-07-17 翻案收窄至此：云同步进 scope（匿名同步码），账号体系仍不做
- 第三方瓦片地图（GCJ-02 瓦片与本项目 WGS 近似坐标存在偏移，且引入外部依赖与配额；零依赖内嵌 SVG 中国地图已交付，足够）
