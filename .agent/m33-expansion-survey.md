## 2. 联程线路候选

优先级含义：P1 是知名度、现有缺口与 schema 可落地性同时较高；P2 是值得收录但存在明显重叠、运营约束或高风险表达成本。候选总数 15，未超过 brief 上限。

### L1 · P1 — 齐鲁文脉·泉城泰山曲阜

- 区域 / 省份：华东 / 山东。
- stops：jinan:2 → taishan:2 → qufu:1；全为现有 id，地理顺序为济南向南到泰安、再向南到曲阜。
- days：建议 [4,5,7]；默认站点合计 5 天，合法可达区间 [3,8]，三档均有效。
- alt / leg：alt:false；通常不需 leg，城市方案可以承接。
- 上海出发：高铁济南进、曲阜东出，预判“直达”；异地还车不是必要条件。
- 季节：春、秋优先，夏季注意泰山夜爬与高温。
- 冲突：不与胶东海滨线重复，仅复用山东大区。
- 证据：山东经典“一山一水一圣人”组合由泰安市官方线路材料直接使用，见 https://tsgw.taian.gov.cn/art/2022/3/7/art_250480_10294198.html 。

### L2 · P1 — 滇西边境·腾冲芒市瑞丽

- 区域 / 省份：西南 / 云南。
- stops：tengchong-heshun:3 → 需新建 mangshi:2 → 需新建 ruili:2；由腾冲向南到芒市、再向西南到瑞丽。
- days：建议 [5,7,10]；默认 7 天，按建议新卡上限计算可达区间 [3,10]，均有效。
- alt / leg：alt:false；不强制 leg，若最终把沿途公路日写成线路专属住宿，则应补 leg。
- 上海出发：腾冲进、瑞丽结束后经芒市机场出；航班与异地还车需实施时实时复核，预判“一次中转”。
- 季节：秋、冬、春；雨季自驾需保守。
- 冲突：腾冲卡可复用，芒市、瑞丽没有现有卡，不落入六张路线型城市卡。
- 证据：云南交通运输厅给出的滇西旅游环线节点含保山、腾冲、芒市、瑞丽，见 https://jtyst.yn.gov.cn/html/2025/xingyexinwen_0218/3133736.html 。

### L4 · P1 — 川藏南线·G318

- 区域 / 省份：西南 / 四川、西藏。
- stops：chengdu:1 → chuanxi-loop:3 → linzhi:3 → lhasa:3；全为现有 id，按成都—康定/川西—林芝—拉萨向西推进。
- days：建议 [10,14]；默认 10 天，合法区间 [4,22]，两档有效。
- alt / leg：alt:true；必须为所有站写 leg，城市卡方案无法表达 G318 沿线多晚落脚与翻越垭口。
- 上海出发：成都进、拉萨出；异地自驾、进藏证件、道路与营运资质约束大，建议合规当地司机，不把普通租车写成默认，难度“折腾”。
- 季节：春末、秋季；雨季塌方和冬季管制必须在实施时核验。
- 冲突：与 route-chuanxi-yading 复用成都/川西，但后半程和产品主题不同；chuanxi-loop 仍作为整体站点，不新增其内部城市，暂不触发迁移。
- 证据：文旅部公开的川藏南线节点顺序为成都—康定—新都桥—理塘—巴塘—左贡—八宿—波密—林芝—拉萨，见 https://zwgk.mct.gov.cn/zfxxgkml/zykf/202012/t20201213_919276.html 。

### L5 · P1（有条件）— 青甘大环线·柴达木敦煌祁连

- 区域 / 省份：西北 / 青海、甘肃。
- stops：chaka-salt-lake:1 → daqaidam-emerald-lake:2 → dunhuang:2 → zhangye:2；全为现有 id。西宁作为进出门户写在线路说明/leg，不虚构为第五站。
- days：建议 [7,10]；默认 7 天，合法区间 [4,13]，两档有效。
- alt / leg：alt:true；必须写 leg，尤其是西宁进出、柴达木公路日和祁连回程不能由城市方案替代。
- 上海出发：西宁往返形成车辆闭环，航班预判“一次中转”；不依赖异地还车。
- 季节：夏、初秋；冬季长距离公路不作为默认。
- 冲突：与 route-qinghai-loop、route-hexi-corridor 重叠较高。只有用户明确需要“一张卡完成著名大环线”时才新增，否则维持两线手工衔接。
- 证据：甘肃文旅官方材料将敦煌、嘉峪关、张掖与青海湖、茶卡、柴达木组合为青甘环线，见 https://www.gswbj.gov.cn/a/2026/04/30/28139.html 。

## 4. 不建议本批新增

- 不再新增一张“川西大环线”：route-chuanxi-yading 与路线型城市卡 chuanxi-loop 已承担主要表达；优先修现有内容或在用户拍板后做迁移，不制造第三份重叠入口。
- 不做“G219 全线”或“G331 全线”：线路跨度、边境证件、季节与补给远超 2–4 站 schema；只保留上面的塔里木、吉林 G331 分段候选。
- 不做“阿里大北线”：高海拔、无人区、落脚点和证件复杂度无法由当前压缩 schema 诚实表达；阿里南线也只列 P2。
- 暂不把根河、额尔古纳、满洲里拆成新城市卡并新建大兴安岭线：这会同时触发路线型 hulunbuir 城市卡与 route-hulunbuir-aershan 的迁移级联，应另立模块。
- 不新增第二张海南线：先把现有 route-hainan-loop 改名为海南东线，或补成真正环岛，避免用重复数据掩盖 F27。
- 青甘大环线仅作“有条件 P1”：若用户不需要一张卡直达著名产品，现有青海小环线 + 河西走廊更符合去重原则。
- 不收海外线路：M11 仍是显式远期待办，本轮不越界。

## 拍板建议

若只做一轮低迁移成本扩容，建议先选 L1、L2、L3、L6、L7、L8、L10、L11，并配套其“需新建”城市卡；L4、L9 属高价值但需要认真写 leg 的第二组；L5、L12、L15 先决定是否接受重叠；L14 必须在用户明确接受高风险内容与维护成本后再开。
